user_account endpoints

Notice

All endpoints that require authentication accept both token from cookies and headers.

Using cookies is a preferred way cause they offer some nice benefits compared to the headers approach: * tokens from cookies are refreshing if are close to expiring, which simplifies the logic of keeping active users logged in.

Whenever you are making a request (everything except ‘GET’), you need to manually include the X-CSRF-TOKEN header, otherwise your requests will be kicked out as invalid too.

Login

POST /user_account/login

Adds auth cookies if login successful

Example request:

{
  "username": "admin",
  "pwd": "1234",
}

Example response:

{
    "msg": "login successful",
    "token": "user_token"
}
Status Codes:

  • 200 OK – no error

  • 401 Unauthorized – Wrong username or password

Registration

POST /user_account/registration

Example request:

{
  "username": "Admin",
  "full_name": "Admin A.A.",
  "email": "t@t.t",
  "phone": null,
  "contacts": null,
  "pwd": "1234"
}

Example response:

{
   "msg": "Check your email...",
}
Status Codes:

  • 200 OK – no error

  • 400 Bad Request – username is not allowed

Reset password request

POST /user_account/reset_password_request

[requires NO auth cookies or headers]

Example request:

{
  "username": "Admin"
}

Example response:

{
   "msg": "Check your email...",
}
Status Codes:

  • 200 OK – no error

  • 400 Bad Request – user not found

  • 403 Forbidden – user is already authenticated

Profile settings

GET /user_account/profile_settings

[requires auth cookies or headers]

Example response:

{
   "contacts": null,
   "email": "d******z@yandex.ru",
   "full_name": "Admin A.A.",
   "is_email_verified": false,
   "phone": null,
   "pwd": "********",
   "username": "Admin"
}
Status Codes:

  • 200 OK – no error

  • 401 Unauthorized – Missing JWT in headers or cookies (Missing Authorization Header; Missing cookie “access_token_cookie”)

  • 403 Forbidden – Token has expired

Confirm email

GET /user_account/confirm_email/(str: token)

Example response:

{
   "msg": "Your email has been confirmed.",
}
Status Codes:

  • 200 OK – no error

  • 400 Bad Request – Wrong token

Reset password

POST /user_account/reset_password/(str: token)

[requires NO auth cookies or headers]

Example request:

{
  "new_pwd": "1234"
}

Example response:

{
   "msg": "Your password has been reset.",
}
Status Codes:

  • 200 OK – no error

  • 400 Bad Request – Wrong token

  • 403 Forbidden – user is already authenticated

Logout

GET /user_account/logout

[requires auth cookies or headers]

Example response:

{
  "msg": "logout successful",
}
Status Codes:

  • 200 OK – no error

  • 401 Unauthorized – Missing JWT in headers or cookies (Missing Authorization Header; Missing cookie “access_token_cookie”)

  • 403 Forbidden – Token has expired
